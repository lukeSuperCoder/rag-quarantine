# 智能问答助手接口联调文档

## 1. 联调目标

本文档定义 Vue3 前端与 Flask 后端的接口契约，覆盖健康检查、文档管理、知识库查询、会话管理、流式问答和引用来源展示。

默认后端地址：

```text
http://127.0.0.1:8000
```

默认前端地址：

```text
http://127.0.0.1:5173
```

所有接口返回 JSON，流式问答接口除外。时间字段统一使用 ISO 8601 字符串。

## 2. 通用响应格式

### 2.1 成功响应

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

### 2.2 失败响应

```json
{
  "code": 4001,
  "message": "参数错误",
  "data": null
}
```

建议错误码：

| code | 说明 |
|------|------|
| 0 | 成功 |
| 4001 | 参数错误 |
| 4002 | 文件类型不支持 |
| 4003 | 资源不存在 |
| 5001 | 文档处理失败 |
| 5002 | Embedding 失败 |
| 5003 | Milvus 操作失败 |
| 5004 | 模型调用失败 |

## 3. 健康检查

### GET `/api/health`

用于确认后端服务、SQLite、Milvus 和 API key 状态。

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "service": "ok",
    "sqlite": "ok",
    "milvus": "ok",
    "zhipu_api_key": "configured"
  }
}
```

## 4. 文档管理接口

### 4.1 上传文档

### POST `/api/documents/upload`

请求类型：`multipart/form-data`

字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file | File | 是 | `.doc`、`.docx`、`.xls` |
| doc_type | string | 否 | 文档类型，不传则后端自动识别 |
| country | string | 否 | 国家或地区 |

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "document_id": "doc_001",
    "filename": "赴日本犬猫检疫要求.docx",
    "status": "uploaded"
  }
}
```

联调要求：

- 前端上传成功后开始轮询文档列表。
- 后端异步或同步执行转换、分块、embedding、入库。
- 如果处理失败，文档状态为 `failed` 并返回 `error_message`。

### 4.2 获取文档列表

### GET `/api/documents`

查询参数：

| 参数 | 必填 | 说明 |
|------|------|------|
| status | 否 | 按状态过滤 |
| keyword | 否 | 按文件名或文档名搜索 |

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "items": [
      {
        "id": "doc_001",
        "filename": "赴日本犬猫检疫要求.docx",
        "doc_name": "赴日本犬猫检疫要求",
        "doc_type": "country_export_req",
        "country": "日本",
        "status": "ready",
        "chunk_count": 8,
        "error_message": "",
        "created_at": "2026-05-21T20:00:00+08:00",
        "updated_at": "2026-05-21T20:02:00+08:00"
      }
    ]
  }
}
```

### 4.3 获取文档详情

### GET `/api/documents/{document_id}`

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "id": "doc_001",
    "filename": "赴日本犬猫检疫要求.docx",
    "doc_name": "赴日本犬猫检疫要求",
    "doc_type": "country_export_req",
    "country": "日本",
    "status": "ready",
    "chunk_count": 8,
    "txt_preview": "赴日本犬猫检疫要求...",
    "error_message": ""
  }
}
```

### 4.4 删除文档

### DELETE `/api/documents/{document_id}`

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "document_id": "doc_001",
    "deleted": true
  }
}
```

联调要求：

- 前端删除前弹出确认框。
- 删除后刷新文档列表和知识库查询结果。
- 后端需要同步删除 SQLite chunk 记录和 Milvus 中对应向量。

## 5. 知识库接口

### 5.1 关键词搜索

### GET `/api/knowledge/search`

查询参数：

| 参数 | 必填 | 说明 |
|------|------|------|
| keyword | 是 | 关键词 |
| doc_type | 否 | 文档类型 |
| country | 否 | 国家或地区 |
| limit | 否 | 默认 20 |

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "items": [
      {
        "chunk_id": "chunk_001",
        "document_id": "doc_001",
        "doc_name": "赴日本犬猫检疫要求",
        "doc_type": "country_export_req",
        "country": "日本",
        "section_title": "狂犬病疫苗",
        "chunk_index": 2,
        "text": "犬猫需接种狂犬病疫苗..."
      }
    ]
  }
}
```

### 5.2 获取 chunk 详情

### GET `/api/knowledge/chunks/{chunk_id}`

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "chunk_id": "chunk_001",
    "document_id": "doc_001",
    "doc_name": "赴日本犬猫检疫要求",
    "section_title": "狂犬病疫苗",
    "chunk_index": 2,
    "text": "完整片段内容..."
  }
}
```

### 5.3 修改 chunk 内容

### PUT `/api/knowledge/chunks/{chunk_id}`

请求：

```json
{
  "text": "修改后的片段内容"
}
```

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "chunk_id": "chunk_001",
    "updated": true,
    "reembedded": true
  }
}
```

联调要求：

- 修改 chunk 后必须重新 embedding 并更新 Milvus。
- 如果 embedding 失败，保留原 chunk 内容和原向量，返回失败信息。

## 7. 流式问答接口

### POST `/api/chat/stream`

请求：

```json
{
  "session_id": "sess_001",
  "question": "宠物入境中国需要隔离多久？",
  "top_k": 5,
  "rerank": true,
  "temperature": 0.7,
  "max_history": 20
}
```

响应类型：`text/event-stream`

### 7.1 SSE 事件格式

#### retrieve_start

```text
event: retrieve_start
data: {"query":"宠物入境中国需要隔离多久？","top_k":5,"rerank":true}
```

#### retrieve_done

```text
event: retrieve_done
data: {"count":5}
```

#### delta

```text
event: delta
data: {"content":"根据"}
```

#### sources

```text
event: sources
data: {"items":[{"chunk_id":"chunk_001","doc_name":"中华人民共和国携带宠物入境检疫要求","doc_type":"entry_regulation","country":"","section_title":"隔离检疫","chunk_index":3,"score":0.82,"text":"相关原文片段..."}]}
```

#### done

```text
event: done
data: {"message_id":"msg_002","session_id":"sess_001"}
```

#### error

```text
event: error
data: {"code":5004,"message":"模型调用失败"}
```

### 7.2 前端处理规则

- 收到 `retrieve_start`：显示检索中状态。
- 收到 `retrieve_done`：显示检索到的来源数量。
- 收到 `delta`：追加到当前助手消息。
- 收到 `sources`：更新右侧引用栏和当前消息引用。
- 收到 `done`：结束 loading，刷新会话列表更新时间。
- 收到 `error`：结束 loading，展示错误信息，保留用户问题。

### 7.3 后端处理规则

- 写入用户消息后再开始检索。
- 模型回答完成后保存助手消息和 sources。
- 如果检索不到资料，仍可调用模型回答，但必须提示未检索到相关资料。
- rerank 异常时不能中断问答，应回退原始向量检索顺序。

## 8. 联调用例

### 8.1 基础健康检查

```bash
curl -sS http://127.0.0.1:8000/api/health
```

预期：

- `service=ok`
- `sqlite=ok`
- `milvus=ok`
- `zhipu_api_key=configured`

### 8.2 创建会话并加载消息

步骤：

1. 调用 `POST /api/chat/sessions` 创建会话。
2. 调用 `GET /api/chat/sessions` 能看到新会话。
3. 调用 `GET /api/chat/sessions/{id}/messages` 返回空消息列表。

### 8.3 流式问答

问题：

```text
宠物入境中国需要隔离多久？
```

预期：

- SSE 返回 `retrieve_start`、`retrieve_done`、多个 `delta`、`sources`、`done`。
- 回答中说明隔离要求。
- sources 至少包含文档名、章节标题、chunk 序号和原文片段。

### 8.4 多轮对话

第一轮：

```text
宠物入境中国需要隔离多久？
```

第二轮：

```text
那需要准备哪些材料？
```

预期：

- 第二轮请求携带同一个 `session_id`。
- 后端读取历史消息并理解“那”指代上一轮主题。
- 两轮消息都能在会话消息列表中查到。

### 8.5 文档上传

步骤：

1. 上传一个 `.docx` 文档。
2. 轮询 `GET /api/documents`。
3. 等待状态变为 `ready`。
4. 使用关键词搜索该文档中的内容。

预期：

- 文档有 `document_id`。
- `chunk_count > 0`。
- 知识库搜索能返回该文档片段。

### 8.6 文档删除

步骤：

1. 删除刚上传的文档。
2. 查询文档列表。
3. 使用相关关键词搜索。
4. 使用相关问题进行 RAG 问答。

预期：

- 文档列表不再显示该文档。
- 知识库搜索不再返回该文档 chunk。
- RAG sources 不再包含该文档。

## 9. 前后端字段对齐清单

| 字段 | 前端用途 | 后端来源 |
|------|----------|----------|
| `document_id` | 文档详情、删除、chunk 归属 | SQLite documents |
| `chunk_id` | 引用定位、片段详情、编辑 | SQLite chunks + Milvus |
| `doc_name` | 展示来源文件 | chunk 元数据 |
| `doc_type` | 过滤和展示文档类型 | chunk 元数据 |
| `country` | 过滤国家/地区 | chunk 元数据 |
| `section_title` | 展示具体段落 | chunk 元数据 |
| `chunk_index` | 展示段落序号 | chunk 元数据 |
| `sources` | 回答引用栏 | RAG 检索结果 |
| `session_id` | 多轮会话记忆 | SQLite chat_sessions |
| `message_id` | 消息定位和刷新 | SQLite chat_messages |

## 10. 验收口径

接口联调完成需要满足：

- 前端能通过健康检查判断后端是否可用。
- 对话页能完成一次完整流式问答。
- 回答内容、引用栏和会话历史三处数据一致。
- 文档上传后能进入知识库查询和 RAG 检索。
- 文档删除后，前端、SQLite 和 Milvus 结果一致。
- 任何接口失败时，前端能展示清晰错误，不出现无限 loading。
