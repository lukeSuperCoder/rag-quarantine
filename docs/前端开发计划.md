# 智能问答助手前端开发计划

## 1. 开发目标

基于 Vue3、Vite、Element Plus 开发一个面向用户使用的宠物检疫政策智能问答系统前端。前端重点展示问答、知识库、文档处理和引用溯源能力，减少教学式说明内容，保持简约、科技感、可操作的产品界面。

前端不直接处理 embedding、Milvus 或模型调用，只通过 Flask API 与后端交互。

## 2. 技术选型

| 模块 | 选型 | 说明 |
|------|------|------|
| 构建工具 | Vite | Vue3 项目开发与构建 |
| UI 框架 | Element Plus | 表单、表格、上传、抽屉、消息提示 |
| 状态管理 | Pinia | 会话、文档、配置状态管理 |
| 路由 | Vue Router | 对话、知识库、文档管理、设置页面 |
| HTTP | Axios | 普通 REST API 调用 |
| 流式响应 | EventSource 或 fetch stream | 接收 SSE 对话流 |
| Markdown | markdown-it | 渲染模型回答 |
| 代码高亮 | highlight.js | Markdown 代码块展示 |

建议前端目录为 `web/`，与现有 Python 脚本目录隔离。

## 3. 页面规划

### 3.1 对话问答页

核心目标：用户能围绕宠物出入境检疫政策连续提问，并看到流式回答和引用来源。

主要区域：

| 区域 | 功能 |
|------|------|
| 左侧会话栏 | 新建会话、切换会话、显示最近会话标题 |
| 中间聊天区 | 展示用户问题、助手回答、流式输出、Markdown 内容 |
| 输入区 | 多行输入、发送按钮、处理中状态、快捷开关 |
| 右侧引用栏 | 展示本轮回答引用的文档、章节、片段内容 |

交互要求：

- 发送问题后立即显示用户消息和助手占位消息。
- 接收到 `delta` 流式事件后增量追加助手回答。
- 接收到 `sources` 事件后刷新引用栏。
- 回答中涉及知识库内容时，引用栏必须展示文件名、章节标题、chunk 序号和原文摘要。
- 支持会话历史切换，切换后重新加载该会话消息。
- 支持 Markdown 渲染，包括列表、表格、加粗和代码块。

### 3.2 文档管理页

核心目标：管理知识库来源文档，支持上传、查看状态、删除和基础编辑。

主要功能：

| 功能 | 说明 |
|------|------|
| 文档上传 | 支持 `.doc`、`.docx`、`.xls`，调用 `/api/documents/upload` |
| 文档列表 | 展示文档名、类型、国家/地区、chunk 数、处理状态、更新时间 |
| 状态展示 | `uploaded`、`converting`、`chunking`、`embedding`、`ready`、`failed` |
| 删除文档 | 删除文档及其对应向量数据 |
| 查看详情 | 查看转换 txt、分块数、错误信息和最近处理记录 |

上传交互要求：

- 上传后不阻塞页面，使用轮询刷新文档处理状态。
- 处理失败时在列表中显示失败原因。
- 删除操作需要二次确认。

### 3.3 知识库查询页

核心目标：让用户通过关键词查看已入库的知识片段，验证知识库内容。

主要功能：

| 功能 | 说明 |
|------|------|
| 关键词搜索 | 调用 `/api/knowledge/search?keyword=...` |
| 条件过滤 | 按 `doc_type`、`country`、`doc_name` 过滤 |
| 片段列表 | 展示命中文本、来源文档、章节标题、chunk 序号 |
| 片段详情 | 抽屉或弹窗展示完整 chunk 内容 |
| 片段编辑 | 修改片段文本后触发重新 embedding |

### 3.4 设置页

核心目标：提供基础问答参数配置，不暴露复杂底层实现。

配置项：

| 配置 | 默认值 | 说明 |
|------|--------|------|
| top_k | 5 | 每轮检索引用片段数量 |
| rerank | false | 是否启用重排序 |
| temperature | 0.7 | 模型回答发散程度 |
| max_history | 20 | 每次问答携带的最大历史消息数 |

设置保存在浏览器本地，并在每次发起问答时传给后端。

## 4. 前端状态设计

### 4.1 chat store

保存对话页状态：

```ts
interface ChatSession {
  id: string
  title: string
  created_at: string
  updated_at: string
}

interface ChatMessage {
  id: string
  session_id: string
  role: 'user' | 'assistant'
  content: string
  sources?: SourceChunk[]
  created_at: string
}
```

### 4.2 document store

保存文档管理状态：

```ts
interface DocumentItem {
  id: string
  filename: string
  doc_name: string
  doc_type: string
  country: string
  status: 'uploaded' | 'converting' | 'chunking' | 'embedding' | 'ready' | 'failed'
  chunk_count: number
  error_message?: string
  created_at: string
  updated_at: string
}
```

### 4.3 source chunk

用于引用栏展示：

```ts
interface SourceChunk {
  chunk_id: string
  doc_name: string
  doc_type: string
  country: string
  section_title: string
  chunk_index: number
  score?: number
  text: string
}
```

## 5. 视觉与交互风格

设计方向：

- 整体色彩使用深浅结合的中性色，辅以蓝绿色科技感强调色。
- 主界面优先展示功能状态，不做大面积营销式说明。
- 对话页保持工具型产品布局，避免过多卡片嵌套。
- 引用来源使用紧凑列表，便于用户核对政策依据。
- 文档管理和知识库查询使用表格、筛选器、抽屉等高密度组件。

关键体验：

- 输入框和消息区域固定高度，消息区内部滚动。
- 流式回答期间发送按钮进入 loading 状态。
- 回答失败时保留用户输入和错误提示，方便重试。
- 长文引用默认折叠，用户点击后查看完整片段。

## 6. 开发阶段

### 第一阶段：项目骨架和对话页

- 创建 `web/` 前端工程。
- 接入 Element Plus、Vue Router、Pinia、Axios。
- 实现基础布局、会话列表、聊天区、输入框。
- 联调 `/api/chat/sessions` 和 `/api/chat/stream`。
- 完成 Markdown 渲染和引用栏展示。

### 第二阶段：文档管理

- 实现文档上传组件。
- 实现文档列表、状态轮询和删除操作。
- 支持文档详情抽屉，展示处理状态、chunk 数和错误信息。

### 第三阶段：知识库查询

- 实现关键词搜索。
- 支持来源过滤、片段详情查看。
- 如后端支持片段编辑，则增加编辑和保存入口。

### 第四阶段：体验打磨

- 增加空状态、错误状态、loading 状态。
- 优化移动端和小屏布局。
- 统一颜色、间距、字体层级。
- 完成前端构建验证和浏览器联调测试。

## 7. 验收标准

- 用户可以在页面中创建会话并完成连续问答。
- 模型回答可以流式展示，且 Markdown 渲染正常。
- 每次回答可以展示引用来源，包含文档名、章节和原文片段。
- 用户可以上传文档，并看到处理状态最终变为 `ready` 或 `failed`。
- 用户可以通过关键词搜索知识库内容。
- 删除文档后，前端列表刷新，后续搜索和问答不再返回该文档内容。
